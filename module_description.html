Module Description Document

Module 1 · Authentication & User Management

Handles all user-related operations including registration, secure login, JWT token issuance, and storage of user profiles. Supports multiple phone numbers per user for multi-contact emergency alerting.

Sub-Modules
  • auth_router.py – FastAPI router exposing /register, /login, /me endpoints
  • auth_service.py – Core logic: password hashing (bcrypt), JWT generation & verification
  • auth_models.py – Pydantic request/response schemas (UserCreate, Token)
  • users.json – Lightweight flat-file user database
Input User credentials (email, password), phone number(s), name
Output JWT access token, authenticated user profile object
Technology FastAPI, Python-Jose (JWT), Passlib (bcrypt), JSON file storage
Frontend Pages Login.jsx, Register.jsx
Module 2 · Camera System

Manages the lifecycle of the webcam, captures video frames in real time, and streams them as MJPEG to the frontend. A dedicated AI pipeline processes each frame for fall detection and overlays bounding boxes and pose landmarks.

Sub-Modules
  • camera_manager.py – Start/stop camera thread, thread-safe singleton
  • camera_ai.py – Per-frame AI inference, alert triggering, MJPEG generation
  • camera_ai_demo.py – Demo/simulation mode (no physical camera required)
Input Webcam device (OpenCV VideoCapture), authenticated user token
Output MJPEG stream feed (/camera/feed), detected events written to shared state
API Endpoints POST /camera/start, POST /camera/stop, GET /camera/status, GET /camera/feed, POST /camera/capture, POST /camera/send_alert
Technology OpenCV, Python threading, FastAPI StreamingResponse, MJPEG protocol
Module 3 · AI Detection Engine

The core intelligence of the system. Analyses video frames using deep learning pose estimation models to detect fall events. Two interchangeable backends are supported — MediaPipe (CPU-friendly) and YOLOv8 (high accuracy). Sensor-based fall detection via accelerometer/gyroscope is also supported.

Sub-Modules
  • yolo_fall_detector.py – YOLOv8 pose model (yolov8n-pose.pt); skeleton angle & velocity analysis
  • pose_fall_detector.py – MediaPipe Pose Landmarker; landmark-based heuristic scoring
  • fall_detector.py – Sensor-based detector (accelerometer + gyroscope threshold logic)
  • audio_detection.py – Breath/sound anomaly detection (stub, extensible)
Input BGR video frame (numpy array) from camera, or accel/gyro vectors from IoT sensors
Output Boolean fall_detected flag, confidence score, annotated frame with landmarks
API Endpoints POST /fall (sensor data), POST /sound
Technology Ultralytics YOLOv8, Google MediaPipe, NumPy, OpenCV
Module 4 · Alert & Escalation System

Implements a 4-level graduated emergency response mechanism that automatically escalates if no acknowledgement is received. Alerts are dispatched through multiple channels (app notification → SMS → voice call → emergency services).

Sub-Modules
  • alert_escalation.py – AlertEscalationManager, AlertEscalation, EscalationLevel enum
  • camera_ai.py – send_sms_alert(), send_email_alert(), send_manual_alert() helpers
  • shared_state.py – Stores active alerts, event log, and fall status
Escalation Levels Level 1 (0 s) – App Notification  |  Level 2 (30 s) – SMS via Twilio  |  Level 3 (60 s) – Voice Call via Twilio  |  Level 4 (120 s) – Emergency Services
Input Detected fall event or manual trigger, authenticated user's registered phone numbers and email
Output SMS messages, email notifications, voice calls; alert log entries in shared state
API Endpoints POST /alerts/escalate, POST /alerts/{id}/respond, POST /alerts/{id}/cancel, GET /alerts/active, GET /alerts/{id}/status
Technology Twilio REST API (SMS + Voice), SMTP (email), Python asyncio, FastAPI background tasks
Module 5 · Dashboard & Real-Time Monitoring

The primary user interface displayed after login. Provides real-time visibility into system status, live camera feed, fall detection events, and alert history. Also exposes analytics (falls per day, alert channel breakdown).

Sub-Modules
  • Dashboard.jsx – Main React page: live MJPEG feed, event log, camera controls, manual alert button
  • AuthContext.jsx – Global authentication context & token management (React Context API)
  • api.js – Axios instance with base URL & interceptors
Input JWT token (stored in localStorage), real-time data from backend REST API
Output Live MJPEG video, system status indicators, event log table, alert history, analytics charts
API Consumed GET /api/dashboard_stats, GET /api/analytics, GET /camera/feed, GET /camera/alerts, POST /camera/send_alert
Technology React 18 (Vite), Axios, Recharts / CSS charts, MJPEG <img> tag streaming
Module 6 · Frontend UI & Theming

Provides the visual framework and immersive aesthetic of the web application. Includes animated backgrounds, security-themed loaders, and a consistent dark-mode design system shared across all pages.

Sub-Modules
  • StreamingBackground.jsx – Animated streaming data background (canvas-based)
  • MatrixBackground.jsx – Matrix-rain canvas animation for login/register screens
  • SecurityLoader.jsx – Animated loading spinner shown during auth transitions
  • App.css / index.css – Global design tokens, dark theme, responsive layout
Input Page load events, component props (colors, speed, density for animations)
Output Rendered animated UI components composited behind page content
Technology React, HTML5 Canvas API, CSS3 animations, Vite build tool
